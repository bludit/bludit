# Security Policy

## Supported Versions

Security fixes are provided for the latest release on the 'main' branch.

## Reporting a Vulnerability

Please report security issues **privately**.

- Preferred: GitHub Private Vulnerability Reporting (Security - "Report a vulnerability") if enabled.
- Otherwise: email to **security@YOURDOMAIN** (or open a GitHub issue only if itâ€™s not sensitive).

Include:

- Summary and details.
- Affected version/commit.
- Impact and reproduction steps.
- Proof of concept.
- Any suggested fix/mitigation.

We aim to respond within **XX hours** and will keep you updated on progress. Please do not disclose publicly until a fix is released or we agree on a disclosure date.
